      • Введение

      JavaScript — это кроссплатформенный объектно-ориентированный язык сценариев. Это небольшой и легкий язык. Внутри хост-среды (например, веб-браузера) JavaScript может быть подключен к объектам его среды, чтобы обеспечить программный контроль над ними.

      JavaScript содержит стандартную библиотеку объектов, таких как Array, Date и Math, а также основной набор языковых элементов, таких как операторы, структуры управления и инструкции. Базовый JavaScript можно расширять для различных целей, дополняя его дополнительными объектами; например:

      * Клиентский JavaScript расширяет основной язык, предоставляя объекты для управления браузером и его объектной моделью документа (DOM).

      * Серверный JavaScript расширяет основной язык, предоставляя объекты, необходимые для запуска JavaScript на сервере.

      В этом руководстве предполагается, что у вас есть следующий базовый опыт:

      * Общее представление об Интернете и Всемирной паутине (WWW).

      * Хорошее знание языка гипертекстовой разметки (HTML).

      * Некоторый опыт программирования. Если вы новичок в программировании, попробуйте одно из руководств по JavaScript, связанных с главной страницей.

      JavaScript и Java

      JavaScript и Java в чем-то схожи, но в чем-то принципиально различаются. Язык JavaScript похож на Java, но не имеет статической типизации Java и строгой проверки типов. JavaScript соответствует большинству синтаксиса выражений Java, соглашениям об именах и базовым конструкциям потока управления, что и стало причиной того, что он был переименован из LiveScript в JavaScript.

      В отличие от системы классов времени компиляции Java, построенной посредством объявлений, JavaScript поддерживает систему времени выполнения, основанную на небольшом количестве типов данных, представляющих числовые, логические и строковые значения. В JavaScript используется объектная модель на основе прототипов вместо более распространенной объектной модели на основе классов. Модель на основе прототипов обеспечивает динамическое наследование; то есть то, что наследуется, может различаться для отдельных объектов. JavaScript также поддерживает функции без каких-либо особых декларативных требований. Функции могут быть свойствами объектов и выполняться как свободно типизированные методы.

      JavaScript — язык очень свободной формы по сравнению с Java. Вам не обязательно объявлять все переменные, классы и методы. Вам не нужно беспокоиться о том, являются ли методы общедоступными, частными или защищенными, и вам не нужно реализовывать интерфейсы. Переменные, параметры и типы возвращаемых функций не типизируются явно.

      Переменные

      Вы используете переменные как символические имена значений в вашем приложении. Имена переменных, называемые идентификаторами, подчиняются определенным правилам.

      Идентификатор JavaScript должен начинаться с буквы, подчеркивания (_) или знака доллара ($); последующие символы также могут быть цифрами (0–9). Поскольку JavaScript чувствителен к регистру, буквы включают символы от «A» до «Z» (прописные) и символы от «a» до «z» (строчные).

      В идентификаторах можно использовать буквы ISO 8859-1 или Unicode, такие как å и ü. Вы также можете использовать escape-последовательности Юникода в качестве символов в идентификаторах. Некоторые примеры допустимых имен: Number_hits, temp99 и _name.

      Объявление переменных

      Вы можете объявить переменную тремя способами:

      С помощью ключевого слова var. Например,

        var x =42 ;

      Этот синтаксис можно использовать для объявления как локальных, так и глобальных переменных.

      Просто присвоив ему значение. Например,

        x =42 ;

      Здесь всегда объявляется глобальная переменная. Он генерирует строгое предупреждение JavaScript. Вам не следует использовать этот вариант.

      С ключевым словом пусть. Например,

        let x =42 ;

      Этот синтаксис можно использовать для объявления локальной переменной области блока. См. раздел Область действия переменной ниже.

      Область действия переменной

      Когда вы объявляете переменную вне какой-либо функции, она называется глобальной переменной, поскольку она доступна для любого другого кода в текущем документе. Когда вы объявляете переменную внутри функции, она называется локальной переменной, поскольку доступна только внутри этой функции.

      JavaScript до ECMAScript 2015 не имеет области действия блочного оператора; скорее, переменная, объявленная внутри блока, является локальной по отношению к функции (или глобальной области видимости), внутри которой находится блок.       Константы

      Типы данных

      Последний стандарт ECMAScript определяет семь типов данных:

      Шесть типов данных, которые являются примитивами

      * Логическое значение: истина и ложь.

      * null: специальное ключевое слово, обозначающее нулевое значение.

      * неопределенный

      * Номер: 42 или 3.14159.

      * Строка: например: «Хауди»

      * Символ (новое в ECMAScript 2015)

      Хотя этих типов данных относительно мало, они позволяют вам выполнять полезные функции с вашими приложениями. Объекты и функции — другие фундаментальные элементы языка. Вы можете думать об объектах как об именованных контейнерах для значений, а о функциях как о процедурах, которые может выполнять ваше приложение.

      оператор if-else

      Используйте оператор if для выполнения оператора, если логическое условие истинно. Используйте необязательное предложение else для выполнения оператора, если условие ложно. Оператор if выглядит следующим образом:

        if(condition) {

        statement_1;

        } else

        {

        statement_2;

        }

      условие может быть любым выражением, которое имеет значение true или false. См. Boolean для объяснения того, что оценивается как true и false. Если условие оценивается как истинное, выполняется оператор_1; в противном случае выполняется оператор_2. Оператор_1 и оператор_2 могут быть любым оператором, включая дополнительные вложенные операторы if.

      Вы также можете объединить операторы, используя else if для последовательной проверки нескольких условий, как показано ниже:

        if(condition_1) {

        statement_1;

        } else if (condition_2)

        {

        statement_2;

        } else

        {

        statement_last; }

      В случае нескольких условий будет выполнено только первое логическое условие, которое имеет значение true. Чтобы выполнить несколько операторов, сгруппируйте их внутри оператора блока ({ ... }). В общем, хорошей практикой всегда является использование блочных операторов, особенно при вложении операторов if:

        if(condition) {

        statement_1_runs_if_condition_is_true;
        statement_2_runs_if_condition_is_true;

        } else

        {

        statement_3_runs_if_condition_is_false;
        statement_4_runs_if_condition_is_false;

        }

      В условном выражении желательно не использовать простые присваивания, поскольку при просмотре кода присваивание можно спутать с равенством. Например, не используйте следующий код:

      if((x=y))

      Пока заявление

      Если условие становится ложным, оператор внутри цикла прекращает выполнение и управление передается оператору, следующему за циклом.

      Проверка условия происходит до выполнения инструкции в цикле. Если условие возвращает true, оператор выполняется и условие проверяется снова. Если условие возвращает false, выполнение останавливается и управление передается следующему оператору while.

      Чтобы выполнить несколько операторов, используйте оператор блока ({ ... }), чтобы сгруппировать эти операторы.

      Пример: Следующий цикл while выполняется до тех пор, пока n меньше трех:

        var n = 0 var x = 0 while ( n < 3

        {
         n++; x+=n;
        }

      На каждой итерации цикл увеличивает n и добавляет это значение к x. Следовательно, x и n принимают следующие значения:

      * После первого прохода: n = 1 и x = 1

      * После второго прохода: n = 2 и x = 3

      * После третьего прохода: n = 3 и x = 6

      После завершения третьего прохода условие n < 3 перестает быть истинным, поэтому цикл завершается.

      Объявления функций

      пределение функции (также называемое объявлением функции или оператором функции) состоит из ключевого слова function, за которым следует:

      * Имя функции.

      * Список аргументов функции, заключенный в круглые скобки и разделенный запятыми.

      * Операторы JavaScript, определяющие функцию, заключенные в фигурные скобки { }.

      Например, следующий код определяет простую функцию с именем Square:

        function square(number)

        {
        return number * number;
        }

      Функция Square принимает один аргумент, называемый числом. Функция состоит из одного оператора, который требует вернуть аргумент функции (то есть число), умноженный на себя. Оператор return указывает значение, возвращаемое функцией.

        return number * number;

      Примитивные параметры (например, числа) передаются функциям по значению; значение передается функции, но если функция меняет значение параметра, это изменение не отражается глобально или в вызывающей функции.

      Оператор переключения

      Используйте этот switch оператор, чтобы выбрать один из множества блоков кода для выполнения.

      * Значение выражения сравнивается со значениями каждого случая.

      * Если есть совпадение, соответствующий блок кода выполняется.

      * Если совпадений нет, выполняется блок кода по умолчанию.
